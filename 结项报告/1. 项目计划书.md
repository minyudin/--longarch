## 一、项目概述

### 1. 项目背景
随着物联网技术在农业领域的推广，传统大棚的人工巡检模式已无法满足现代化种植对数据实时性和准确性的需求。本项目依托三个实体大棚场景，搭建一套集数据采集、存储、可视化于一体的信息管理系统，实现大棚环境的远程监控与数据化管理。

### 2. 项目目标
- 打通硬件传感器到数据库的数据链路，支持多大棚、多点位、多类型传感器的统一接入
- 提供可视化大屏展示大棚实时环境数据与历史趋势
- 实现设备管理、用户权限、数据查询等后台管理功能

### 3. 开发环境与条件
- **开发工具**：IntelliJ IDEA、VS Code、Navicat
- **数据库**：MySQL 8.0
- **运行环境**：JDK 17、Node.js 18、Mosquitto MQTT Broker
- **版本管理**：Git + GitHub

## 二、需求分析

### 1. 目标用户与使用场景
- **大棚管理员**：通过 PC 端大屏查看实时传感器数据、管理设备
- **农场数据分析员**：查看整体运营数据、历史趋势、导出报表
- **系统管理员**：管理各类数据

### 2. 核心功能清单
- 大棚与点位管理（增删改查）
- 传感器设备管理与在线状态监控
- 实时数据采集（MQTT）与历史数据存储
- 可视化大屏（ECharts 图表、实时折线图）
- 数据导出与查询

### 3. 非功能需求
- **性能要求**：支持 100+ 传感器并发上报，页面响应时间 < 1s
- **安全要求**：用户密码加密、接口 JWT 鉴权、MQTT 用户名密码认证
- **兼容性要求**：Chrome / Edge 现代浏览器，响应式布局

---

## 三、技术路线

### 1. 总体架构
采用**前后端分离 + 物联网接入**的三层架构：
- **感知层**：传感器 → 主控网关 → MQTT 上报
- **服务层**：Spring Boot 后端订阅 MQTT，解析入库，提供 REST API
- **展示层**：Vue 3 前端调 API + WebSocket 实时推送，ECharts 可视化

### 2. 主要技术
- **前端**：Vue 3 + Vite + Element Plus + ECharts + Pinia
- **后端**：Spring Boot 3 + MyBatis-Plus + Spring Security + JWT
- **数据库**：MySQL 8.0
- **物联网**：Mosquitto MQTT Broker + paho-mqtt 客户端

### 3. 数据设计初步考虑
- **主要实体**：大棚（greenhouse）、点位（plot）、传感器设备（sensor_device）、传感器数据（sensor_data）、用户（user）、角色（role）
- **关键关系**：
  - 大棚 1:N 点位，点位 1:N 传感器设备
  - 传感器设备 1:N 传感器数据（按时间存储）
