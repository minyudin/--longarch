# Overview 页面设计思维导图

## 零、核心认知：点位制

每个物理"点位"对应数据库中一个 `plot` 记录。3个大棚共 **13个点位**，每个大棚1块大屏。
大屏通过 `screen_token` 绑定到**大棚级别**，一次性展示本棚所有点位数据。

```
稻梦田园
├── 1号大棚（大屏 SCR-DM01）
│   ├── 1#点位（棚上桁架）→ 球形摄像机 + 环境传感器(温度/湿度/光照/CO2)
│   ├── 2#点位（棚壁）    → NPK传感器 + pH传感器 + 土温土湿传感器
│   ├── 3#点位（棚壁）    → NPK传感器 + pH传感器 + 土温土湿传感器
│   ├── 4#点位（棚壁）    → NPK传感器 + pH传感器 + 土温土湿传感器
│   └── 5#点位（棚壁）    → NPK传感器 + pH传感器 + 土温土湿传感器
│
├── 2号大棚（大屏 SCR-DM02）
│   ├── 6#点位（棚上桁架）→ 球形摄像机 + 环境传感器(温度/湿度/光照/CO2)
│   ├── 7#点位（棚壁）    → NPK传感器 + pH传感器 + 土温土湿传感器
│   └── 8#点位（棚壁）    → NPK传感器 + pH传感器 + 土温土湿传感器
│
└── 3号大棚（大屏 SCR-DM03）
    ├── 9#点位（棚上桁架） → 球形摄像机 + 环境传感器(温度/湿度/光照/CO2)
    ├── 10#点位（棚壁）   → NPK传感器 + pH传感器 + 土温土湿传感器
    ├── 11#点位（棚壁）   → NPK传感器 + pH传感器 + 土温土湿传感器
    ├── 12#点位（棚壁）   → NPK传感器 + pH传感器 + 土温土湿传感器
    └── 13#点位（棚壁）   → NPK传感器 + pH传感器 + 土温土湿传感器
```

### 点位分类

| 类型 | 包含设备 | 数量 | 特征 |
|------|----------|------|------|
| 环境点位 | 球形摄像机 + 温湿度/光照/CO2 | 每棚1个（1#/6#/9#） | 挂在桁架上，有摄像头+环境数据 |
| 土壤点位 | NPK + pH + 土温土湿 | 1号棚4个、2号棚2个、3号棚4个 | 埋在土里，只有土壤数据 |

### 关键差异

- **不同大棚的土壤点位数量不同**：1号棚4个、2号棚2个、3号棚4个 → 前端必须动态渲染
- **每个土壤点位的传感器种类固定**：NPK(3指标) + pH(1指标) + 土温土湿(2指标) = 共6个指标
- **环境点位传感器是 4合1**：一个设备上报4种指标（temperature/humidity/light/CO2）
- **先期每类只部署1套**：实际初期可能只有部分传感器在线，前端必须处理"无数据"状态

## 一、数据模型变更（需要后端配合）

### 当前问题

当前 `screen_device.plot_id` 绑定的是单个 plot。如果改为点位制（13个plot），
大屏需要看到同一大棚下**所有点位**的数据，而不是单个 plot 的数据。

### 方案：后端 API 改造

```
方案A（推荐）：screen_device 绑定大棚，新增 greenhouse_id 概念
  screen_device.greenhouse_id → 关联多个 plot
  API 返回该大棚下所有 plot 的聚合数据

方案B：screen_device 保持绑定单个"主点位"(环境点位)
  API 返回时自动查同一大棚下所有相关 plot 的数据
  需要 plot 表新增 greenhouse_id 或 parent_plot_id 字段来分组

方案C：前端多次请求
  大屏配置多个 token 或多个 plotId
  前端分别请求每个点位的数据再聚合
  （不推荐，增加复杂度）
```

## 三、子组件清单

### 1. DashboardHeader

| Prop | 类型 | 说明 |
|------|------|------|
| greenhouseName | string | 大棚名（如 "1号大棚"） |
| farmName | string | 农场名（如 "稻梦田园"） |
| updatedAt | string | 最近数据时间 |
| cameraStatus | string \| null | 摄像头在线状态 |

### 2. CameraPanel

| Prop | 类型 | 说明 |
|------|------|------|
| camera | CameraInfo \| null | 摄像头信息 |

**状态场景**：

| 场景 | 条件 | UI |
|------|------|-----|
| 正常播放 | camera != null && status == "online" | flv.js 播放 |
| 摄像头离线 | camera != null && status != "online" | 离线占位 + 状态文字 |
| 无摄像头 | camera == null | "未绑定摄像头" 占位 |
| 播放失败 | flv.js error event | "视频流连接失败" + 自动重连 |
| 加载中 | data == null | 骨架屏 |

### 3. EnvCards

| Prop | 类型 | 说明 |
|------|------|------|
| environment | Record<string, number> \| undefined | 环境指标 map |

**指标映射**（前端维护映射表，key 来自后端 sensor_type）：

| key | 显示名 | 单位 | 正常范围 |
|-----|--------|------|----------|
| temperature | 温度 | ℃ | 15~35 |
| humidity | 湿度 | % | 40~90 |
| light | 光照 | lux | 0~100000 |
| CO2 | CO₂ | ppm | 200~1000 |

**状态场景**：

| 场景 | 条件 | UI |
|------|------|-----|
| 正常 | 有值 && 在范围内 | 绿色数值 |
| 告警 | 有值 && 超范围 | 红色/橙色高亮 + 脉冲动效 |
| 无数据 | key 不存在 或 值为 null | 显示 "--" |
| 传感器离线 | 环境传感器 status != online | 全部灰色 + "传感器离线" 标记 |
| 加载中 | data == null | 骨架屏 |

### 4. PlotGrid（原 SoilGrid，改名更准确）

| Prop | 类型 | 说明 |
|------|------|------|
| plots | PlotPointInfo[] | 土壤点位列表（动态长度） |

每个 PlotPointInfo:
```
{
  plotId: number
  plotName: string        // "2#点位"
  sensors: Record<string, number>  // { N:3.1, P:1.8, K:5.2, pH:6.8, soilTemp:22.5, soilMoisture:65.3 }
  lastSampleAt: string | null
}
```

**土壤指标映射**：

| key | 显示名 | 单位 |
|-----|--------|------|
| N | 氮(N) | mg/kg |
| P | 磷(P) | mg/kg |
| K | 钾(K) | mg/kg |
| pH | pH值 | — |
| soilTemp | 土壤温度 | ℃ |
| soilMoisture | 土壤湿度 | % |

**状态场景**：

| 场景 | 条件 | UI |
|------|------|-----|
| 正常 | plots.length > 0 | 动态网格，4个点位→4列，2个点位→2列(更宽) |
| 部分离线 | 某点位 sensors 为空 或 lastSampleAt 很久前 | 该卡片灰色 + "数据过期" |
| 全部无数据 | plots.length == 0 | "暂无土壤监测点位" |
| 加载中 | data == null | 骨架屏 |

## 四、全局状态 & 错误处理

```
useOverview() hook — 10秒轮询
│
├── 正常: data 有值 → 渲染所有子组件
│
├── 首次加载中: loading=true, data=null
│   └── 全局骨架屏（保持 Matrix 数字雨背景）
│
├── API 失败:
│   ├── 401 token 无效 → "无效的大屏 Token，请联系管理员配置"
│   ├── 404 大棚不存在 → "未找到关联大棚"
│   ├── 网络断开 → "无法连接服务器，自动重试中..."
│   └── 其他 → 显示 error.message
│
├── 轮询容错:
│   ├── 有旧 data + 本次失败 → 继续显示旧数据 + 顶部小红条提示
│   ├── 有旧 data + 本次成功 → 更新数据，清除错误
│   └── 永远不清空已有 data（除非 token 变了）
│
└── Token 来源:
    ├── URL: ?token=tk-dm01-a1b2c3d4e5f6 (生产)
    └── .env: VITE_DASHBOARD_TOKEN (开发)
```

## 六、待决定

### 1. 后端 API 改造（必须）

当前 `screen_device.plot_id` 只绑定1个 plot。改为点位制后需要：
- **plot 表新增 `greenhouse_id` 或 `parent_id` 字段**，用于分组同一大棚下的所有点位
- **screen_device 改为绑定 greenhouse 级别**，或通过 plot 的 parent 关系查出同棚所有点位
- **API 响应结构**改为返回 `plots[]` 数组而非单个 `soil[]`

### 2. 管理端创建 13 个点位

需要在管理端执行：
- 创建13个 plot 记录（1#~13#点位）
- 每个点位绑定对应的传感器设备
- 环境点位(1#/6#/9#)额外绑定摄像头
- 更新 screen_device 的绑定关系

### 3. 指标映射表放哪

- **推荐**：前端维护映射表（sensor_type → 中文名/单位/图标/范围）
- 原因：指标种类固定（环境4种+土壤6种），后端已通过 sensor_type 区分，无需额外接口
- 映射表不硬编码具体值，只做 key→显示信息 的转换

### 4. 视频流

- 摄像头 RTMP 推流到 SRS → SRS 转 HTTP-FLV / HLS
- 大屏用 flv.js 播 HTTP-FLV（延迟低）
- 需确认 SRS 服务是否已部署
