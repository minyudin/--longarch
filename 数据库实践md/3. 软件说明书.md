## 一、项目概述

### 1. 项目背景

随着物联网技术在农业领域的推广，传统大棚的人工巡检模式已无法满足现代化种植对数据实时性和准确性的需求。本项目依托三个实体大棚场景，搭建一套集数据采集、存储、可视化于一体的信息管理系统，实现大棚环境的远程监控与数据化管理。

### 2. 系统目标（完成情况）

| 目标 | 实现说明 |
|------|----------|
| 传感器到数据库的数据链路 | 已实现：MQTT（Mosquitto）上报 → Spring Boot 订阅解析 → MySQL 持久化；支持按设备号、遥测/心跳/回调等主题分流处理 |
| 可视化大屏与历史趋势 | 已实现：PC 管理端工程集成 ECharts（Vue 或 React 版本见下），大屏与仪表盘展示环境与设备数据 |
| 设备管理与权限 | 已实现：后台维护大棚/点位/设备及用户角色；接口层 Sa-Token + Redis |
| 微信小程序端认养与作业协同 | 已实现：Taro 小程序对接同一 REST 后端；认养、任务创建、操作员审核与责任域与库表 `operation_task`、`operator_plot_binding` 等一致 |

## 三、系统设计

### 1. 系统架构

采用 **前后端分离 + MQTT 物联网接入**：

- **感知层**：传感器/网关经 MQTT 上报遥测、心跳等。  
- **服务层**：`longarch-server`（Spring Boot）订阅 MQTT，解析入库；控制器普遍使用 **`/api/v1`** 前缀（`context-path` 为 `/`，端口默认 **8081**，见 `application.yml`）。接口文档：**Knife4j / SpringDoc** 已启用，浏览器通常访问 **`http://<主机>:8081/doc.html`**（亦已将 `/doc.html` 等纳入 Sa-Token 匿名放行，见 `SaTokenConfig`）。  
- **展示层**：PC 端 **`longarch-admin`**（Vue）或 **`longarch-admin-next`**（React，联调常用）；**独立大屏 **`bridge-viz`**（会议室 / 棚头全屏展示，Token 鉴权）；移动端 **`miniapp`**（Taro）。设备侧 **MQTT → 后端 → MySQL**；管理端与大屏经 **HTTP** 拉数；仪表盘侧还可结合 **`MqttMessageBuffer`** 等缓冲 MQTT 消息摘要供调试类接口展示。

```java
// longarch-server/.../MqttConfig.java（节选）
@Bean
public MqttPahoClientFactory mqttClientFactory() {
    DefaultMqttPahoClientFactory factory = new DefaultMqttPahoClientFactory();
    MqttConnectOptions options = new MqttConnectOptions();
    options.setServerURIs(new String[]{mqttProperties.getBrokerUrl()});
    if (mqttProperties.getUsername() != null && !mqttProperties.getUsername().isBlank()) {
        options.setUserName(mqttProperties.getUsername());
    }
    if (mqttProperties.getPassword() != null && !mqttProperties.getPassword().isBlank()) {
        options.setPassword(mqttProperties.getPassword().toCharArray());
    }
    options.setAutomaticReconnect(true);
    options.setCleanSession(false);
    factory.setConnectionOptions(options);
    return factory;
}
```

public void handleCallback(Message<?> message) {

## 六、用户手册

### 1. 安装与部署

**本地联调（Windows，路径以本机为准）** — 详见仓库根目录 **`启动命令.md`**，典型顺序：

1. 启动 Mosquitto（指定项目内 `mosquitto-dev.conf` 等）。  
2. 启动 MySQL、Redis（按 `application-dev.yml` 或本地习惯）。  
3. 后端：`cd longarch-server`，`mvn spring-boot:run`（或 IDE 运行）。  
4. PC 管理端：**联调推荐** `cd longarch-admin-next` 后 `pnpm dev`（见 **`启动命令.md`**，默认 `5173`）；若使用 Vue 版则 `cd longarch-admin` 后 `npm run dev`。  
5. **可视化大屏（可选，演示大棚实时大屏时启动）**：`cd bridge-viz`，`pnpm install`，`pnpm dev`，浏览器访问 **`http://127.0.0.1:5174`**（Vite 已将 **`/api`** 代理到后端 `8081`）。生产环境请配置 **`VITE_API_BASE`** 或通过反向代理使大屏与后端同源，避免跨域与混合内容问题。  
6. 小程序：`cd miniapp`，`pnpm install`，`pnpm build:h5` + `pnpm preview:h5` 或微信开发者工具导入构建产物；配置合法 **`TARO_APP_API_BASE`** 与小程序合法域名。  

**Docker**：见 **`docs/Docker部署.md`**，分别在 **`longarch-server`**、**`longarch-admin`** 目录构建镜像；**`bridge-viz`** 需单独 `pnpm build` 后用 Nginx 托管静态资源（或与现有站点同域部署）。  

**生产裸机 Linux**：参考 **`deploy/deploy.sh`**、**`deploy/nginx-longarch.conf`**、**`deploy/longarch.service`**，环境变量模板 **`deploy/.env.example`**。

### 2. 操作指南

#### 2.1 管理员（PC 管理端）

1. 浏览器打开管理端（联调一般为 **`http://127.0.0.1:5173`**），使用管理员账号登录。  
2. **地块 / 大棚**：在「地块」相关页面维护大棚与子点位；**大棚级地块至少再绑定一个子点位**后，才会出现在独立大屏的大棚切换列表中（后端按规则过滤空大棚）。  
3. **绑定大屏（首次授权）**：在地块行操作中选择 **「绑定大屏」**，填写大屏名称（及可选设备编号），提交后系统返回 **大屏 Token** — 该 Token 对应一条大屏设备记录，用于 **`bridge-viz`** 访问 **`/api/v1/screen/**`**。  
4. **大屏集中管理**：进入侧边栏菜单 **「大屏」 / Screens**（路由 **`/screens`**），可查看全部大屏、**复制 Token**、**复制激活链接**（大屏浏览器打开链接可自动写入本地存储）、**重新生成 Token**（旧 Token 立即失效）、删除大屏等。  

#### 2.2 独立可视化大屏（`bridge-viz`，会议室 / 棚头电脑）

1. 确保 **后端已启动**（`8081`），且 MQTT → 入库链路正常，否则大屏只有框架数据或为空。  
2. 开发环境：启动 **`bridge-viz`** 后访问 **`http://127.0.0.1:5174`**。若本地从未保存过 Token，会进入 **`/setup` 激活页**。  
3. **激活方式（任选）**：  
   - 向运维索要 **激活链接**（管理端「大屏」页复制），用大屏用的浏览器打开，可自动保存 Token 并跳转；  
   - 或在激活页 **手动粘贴 Token**（与 **`X-Screen-Token`** 一致），通过 **`GET /api/v1/screen/greenhouses` 校验** 后写入本地存储。  
4. 进入总览后可通过 **大棚切换器** 切换同一农场下不同大棚视角；图表依赖后端 **`overview` / `env-history`** 等接口返回。  
5. Token 失效（被重新生成或删除）时，需重新在 **`/setup`** 完成激活。

#### 2.3 数据分析员、小程序用户

| 角色 | 操作 |
|------|------|
| 数据分析员 | 管理端登录 → 使用统计与查询、导出（以界面开放功能为准） |
| 认养者 | 打开微信小程序 → 登录 → 认养/地块浏览 → 发起任务 → 查看进度 |

### 3. 接口与配置查阅
